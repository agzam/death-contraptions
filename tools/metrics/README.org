#+title: metrics

Search and query Grafana metrics (e.g. metrics.qlikcloud.io): run PromQL, search
dashboards, read panel queries. Borrows the session cookie from Brave on macOS,
like the splunk tool, and auto-opens the login when the session is stale.

macOS only.

* Tools

- =metrics-query= - run a PromQL query (range or instant) through a datasource.
- =metrics-list-datasources= - list Grafana datasources (uid, name, type).
- =metrics-search-dashboards= - search dashboards by text.
- =metrics-get-dashboard= - show a dashboard's panels and their PromQL.
- =metrics-metric-names= - list or substring-search metric names.

* Prerequisites

- macOS (uses Keychain to decrypt Brave cookies).
- Brave browser, logged into the Grafana tenant.

* Auto-login

The first call probes =GET /api/user=. If the session is missing or expired, the
server runs =open -a "Brave Browser" https://<host>= and polls for up to
=:login-timeout-ms= until auth succeeds. When the upstream SSO session is still
valid this is silent; otherwise finish the SSO/MFA prompt once and the call
proceeds. Disable with =:auto-login? false=.

* Configuration

Settings live in =config.edn= (generated by =setup.bb= from =local-config.edn=):

- =:host= - Grafana hostname (e.g. =metrics.qlikcloud.io=).
- =:auto-login?= - auto-open the browser on a stale session (default =true=).
- =:login-timeout-ms= - bootstrap poll budget in ms (default =30000=).
- =:browser-app= - macOS app to open for login (default ="Brave Browser"=).

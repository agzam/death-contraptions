#+title: ECA MCP Servers

Babashka scripts implementing [[https://modelcontextprotocol.io/][MCP]] (Model Context Protocol) over stdin/stdout
JSON-RPC. Each server exposes tools that the AI agent calls during a session.

All servers share the same skeleton: read JSON lines from stdin, dispatch via
=handle-request=, write JSON response to stdout. No external API keys needed -
auth is bootstrapped from local app state (browser cookies, desktop app
credentials, running processes).

Configured in =~/.config/eca/config.json= under =mcpServers=.

* elisp-eval-server.bb

Evaluate Emacs Lisp in a running Emacs server via =emacsclient=.

** Tools

=elisp-eval= - Execute arbitrary elisp. Code is written to a temp file and
wrapped in a harness that captures the return value, new =*Messages*=,
=*trace-output*=, and backtraces into separate temp files - avoiding all shell
escaping issues.

| Param        |    Default | Description                                                    |
|--------------+------------+----------------------------------------------------------------|
| =code=         | (required) | Elisp to evaluate                                              |
| =timeout=      |         30 | Seconds before kill. Increase for profiling, =sit-for=, installs |
| =print_length= |        200 | Max list/vector elements printed. -1 for unlimited             |
| =print_level=  |         10 | Max nesting depth printed. -1 for unlimited                    |

The result is read from a temp file (with =print-length= / =print-level= in
effect) rather than stdout, so output is always bounded. Stdout is a fallback.
Output is truncated at 50K chars.

Backtraces use =handler-bind= (Emacs 29+) to capture the stack before it
unwinds, with a fallback for older Emacs.

=emacs-screenshot= - Capture the Emacs frame as a PNG image (base64).

| Platform       | Method                                                      |
|----------------+-------------------------------------------------------------|
| macOS (ns)     | Swift + CoreGraphics to find CGWindowID, then =screencapture= |
| X11            | =outer-window-id= frame parameter + ImageMagick =import=        |
| Wayland (pgtk) | =gnome-screenshot -w=, falls back to =grim=                     |

** Dependencies

- =emacsclient= (Emacs server must be running)
- macOS screenshot: =swift=, =screencapture=
- X11 screenshot: =import= (ImageMagick)
- Wayland screenshot: =gnome-screenshot= or =grim=

* jxa-browser-server.bb

Control and inspect the default browser on macOS via JavaScript for Automation
(JXA). Auto-detects the default HTTPS handler from LaunchServices.

** Tools

| Tool                    | Description                                                                       |
|-------------------------+-----------------------------------------------------------------------------------|
| =browser-list-tabs=       | List all tabs across all windows (JSON array)                                     |
| =browser-read-active-tab= | Read page as text; optional =query= greps with context; =format=html= for raw HTML    |
| =browser-get-selection=   | Get currently selected text                                                       |
| =browser-navigate=        | Go to URL, back, forward, reload. Waits for page load                             |
| =browser-click=           | Click by CSS selector or visible text (case-insensitive substring match)          |
| =browser-type=            | Type into inputs/contenteditable. Supports =clear=, =submit=                          |
| =browser-query=           | DOM query - returns tag, text, attributes, bounding rect, visibility              |
| =browser-execute-js=      | Run arbitrary JS. Supports same-origin iframes. Base64-encoded to avoid escaping  |
| =browser-tab=             | Switch (by index or URL regex), open, close tabs                                  |
| =browser-screenshot=      | Browser window screenshot as base64 PNG                                           |
| =browser-network=         | Start/get/stop network capture. Patches fetch + XHR. Reads Resource Timing API    |
| =browser-console-logs=    | Start/get/stop console capture. Patches console.log/warn/error/info/debug         |
| =browser-wait=            | Poll for: element visible/hidden/present, text on page, URL regex. 500ms interval |
| =browser-scroll=          | Scroll to element, top/bottom, or by pixel amount                                 |

** Implementation notes

- JS execution writes code to a temp file and reads it via JXA's =Path()= to
  avoid encoding issues across the osascript bridge.
- HTML-to-text uses macOS =textutil=.
- Click dispatches a full synthetic event chain: scrollIntoView, mouseover,
  mousedown, mouseup, click.
- Navigate polls =document.readyState= for up to 9 seconds.
- Screenshot uses Swift + CoreGraphics (same approach as elisp-eval).

** Dependencies

macOS only: =osascript=, =plutil=, =jq=, =textutil=, =swift=, =screencapture=.

* slack-server.bb

Search Slack messages and fetch threads. Zero-config auth - extracts
credentials directly from the Slack desktop app.

** Tools

=slack-search= - Search messages using Slack search operators (=from:=, =in:=,
=before:=, =after:=, =has:link=, =has:reaction=, etc.).

| Param     | Default     | Description                                      |
|-----------+-------------+--------------------------------------------------|
| =query=     | (required)  | Search query                                     |
| =workspace= | env default | Workspace name (e.g. =myteam= or =myteam.slack.com=) |
| =count=     | 20          | Results per page                                 |
| =page=      | 1           | Page number                                      |

=slack-fetch-thread= - Fetch a full thread (all replies) from a Slack message
URL. Falls back to =conversations.history= if =conversations.replies= returns
nothing. Resolves =<@U123>= mentions to display names.

** Auth mechanism

1. Discovers workspace tokens from the Slack desktop app's =storage/root-state.json=
   (falls back to scanning IndexedDB with =rg=)
2. Copies the Slack SQLite cookie DB, extracts the encrypted =d= cookie
3. Derives AES key via PBKDF2 (SHA1, 1003 iterations, 16-byte key from
   Keychain "Slack Safe Storage")
4. Decrypts AES-128-CBC, strips PKCS7 padding
5. Fetches workspace HTML with the =d= cookie, scrapes =api_token= (=xoxc-...=)
6. Auto-refreshes on token expiry (throttled to once per 60s)

** Environment variables

| Var                    | Default                              | Description                 |
|------------------------+--------------------------------------+-----------------------------|
| =SLACK_DEFAULT_HOST=     | (none)                               | Default workspace hostname  |
| =SLACK_DATA_DIR=         | =~/Library/Application Support/Slack/= | Slack app data directory    |
| =SLACK_KEYCHAIN_SERVICE= | =Slack Safe Storage=                   | macOS Keychain service name |

** Dependencies

macOS: =security= (Keychain), =sqlite3=, =openssl=, =rg= (ripgrep).
Linux: would need adaptation for cookie storage and keychain.

* splunk-server.bb

Run SPL queries against Splunk. Borrows session credentials from the Brave
browser's cookie store.

** Tools

=splunk-search= - Execute SPL queries. Auto-prepends =search= keyword for
non-piped queries.

| Param         | Default    | Description                                 |
|---------------+------------+---------------------------------------------|
| =query=         | (required) | SPL query string                            |
| =earliest_time= | -24h       | Start of time range (Splunk time modifiers) |
| =latest_time=   | now        | End of time range                           |
| =max_results=   | 100        | Max results (up to 10000)                   |

=splunk-indexes= - List available indexes with event counts and sizes. Optional
=include_internal= flag for =_=-prefixed indexes.

=splunk-search-metadata= - Discover hosts, sources, sourcetypes for an index
(top 50 by count). Uses =| metadata= SPL command.

** Auth mechanism

Same Chromium v10 cookie decryption as the Slack server, but targeting Brave:

1. Copies Brave's SQLite cookie DB
2. Derives AES key from "Brave Safe Storage" Keychain entry
3. Decrypts the Splunk session cookie (=splunkd_<port>=)
4. Also extracts the CSRF token cookie
5. Routes requests through the Splunk web proxy path
6. Auto-refreshes on 401/403 (throttled to once per 60s)

** Environment variables

| Var         | Default    | Description              |
|-------------+------------+--------------------------|
| =SPLUNK_HOST= | (required) | Splunk instance hostname |

** Dependencies

macOS: =security= (Keychain), =sqlite3=, =openssl=.
Linux: would need adaptation for Chromium cookie storage and keychain.

* k8s-server.bb

Manage Kubernetes clusters via kubectl, stern, and helm. Delegates to CLI tools
for all operations; adds output cleaning (kubectl-neat), secrets masking, and
stateful watch/diff via cached snapshots.

** Tools

| Tool                  | Delegates to     | Description                                                        |
|-----------------------+------------------+--------------------------------------------------------------------|
| =k8s-contexts=          | kubectl config   | List available kubeconfig contexts                                 |
| =k8s-resources-list=    | kubectl + neat   | List resources of any kind. Cleaned JSON, secrets redacted         |
| =k8s-resources-get=     | kubectl + neat   | Get specific resource. Cleaned JSON, secrets redacted              |
| =k8s-resources-apply=   | kubectl apply    | Create or update from YAML/JSON manifest                          |
| =k8s-resources-delete=  | kubectl delete   | Delete a resource                                                  |
| =k8s-pod-logs=          | stern / kubectl  | Multi-pod log tailing with regex. Falls back to kubectl if no stern |
| =k8s-pod-exec=          | kubectl exec     | Execute command in a pod container                                 |
| =k8s-events=            | kubectl          | List events, optionally filtered by resource                       |
| =k8s-describe=          | kubectl describe | Human-readable resource description with events                    |
| =k8s-api-resources=     | kubectl          | List available API resource types including CRDs                   |
| =k8s-helm-list=         | helm list        | List Helm releases                                                 |
| =k8s-helm-status=       | helm status      | Get Helm release status                                            |
| =k8s-resources-watch=   | kubectl + diff   | Stateful watch: first call snapshots, subsequent calls return diff |

All tools accept optional =kubeconfig= and =context= params for multi-cluster
support. Most also accept =namespace= and =all_namespaces=.

** Key features

- Output cleaning: pipes =kubectl -o json= through =kubectl-neat= to strip managed
  fields, defaults, and runtime noise (~60-80% reduction)
- Secrets masking: =.data= and =.stringData= values in Secret resources are replaced
  with =***REDACTED***= before returning to the LLM
- Watch/diff: =k8s-resources-watch= caches resource lists in an atom. Subsequent
  calls diff against the cached state and return only additions, removals, and
  modifications with elapsed time
- Log tailing: uses =stern= for multi-pod regex matching with =--no-follow= for
  bounded output. Falls back to =kubectl logs= if stern is not installed
- Output truncation: all results capped at 50K chars

** Dependencies

Required:
- =kubectl=

Optional (recommended):
- =stern= (=brew install stern=) - multi-pod log tailing with regex
- =helm= (=brew install helm=) - Helm release inspection
- =kubectl-neat= - strips managed fields from resource output. Install from
  [[https://github.com/itaysk/kubectl-neat/releases][GitHub releases]] or place binary on PATH

* Development

** Adding a new server

1. Create =your-server.bb= in this directory
2. Implement the MCP protocol: =initialize=, =notifications/initialized=,
   =tools/list=, =tools/call=
3. Add to =~/.config/eca/config.json= under =mcpServers=
4. Restart via =mcp/stopServer= + =mcp/startServer= (see elisp-eval tool description)

** Linting

#+begin_src sh
clj-kondo --lint your-server.bb
#+end_src

** Testing manually

#+begin_src sh
echo '{"jsonrpc":"2.0","id":1,"method":"initialize","params":{}}' | bb your-server.bb
echo '{"jsonrpc":"2.0","id":2,"method":"tools/list","params":{}}' | bb your-server.bb
#+end_src
